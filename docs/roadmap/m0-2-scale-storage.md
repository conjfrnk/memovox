> **Wave:** 0 ┬╖ **Effort:** XL ┬╖ **Status:** not started

## Goal
Make the free, stdlib-only store survive a real-sized corpus without changing a single answer. This track does three things, in order: (1) extracts `VectorIndex` / `LexicalIndex` / `GraphStore` abstract interfaces out of `LoomStore`, mirroring the model-backend registry in `src/memovox/backends/`, with today's SQLite implementation as the always-available free default тАФ a **pure no-op refactor first**, locked by a byte-identical parity gate; (2) kills the brute-force per-row cosine on the free retrieval path with unit-normalized stored vectors + batched `struct` scoring, plus an optional (default-OFF) FTS5 candidate prefilter; (3) makes cross-corpus consolidation **incremental and observable** тАФ a `consolidation_watermark` meta key, scan new-claims-vs-**all** (never new-vs-new), and replace the silent `max_claims=600` truncation in `find_contradictions`/`cluster_claims` with reported paging (`claims_scanned` / `claims_skipped` / `capped` in `ConsolidationReport`). Optional ANN (LanceDB/Qdrant), K├╣zu graph, and Tantivy+SPLADE lexical backends land behind `is_available()` as opt-in, never in the CI gate.

## Why it matters
The spec's free-path promise is "you can `sqlite3 ~/.memovox/memovox.db` and read your knowledge directly" (see `src/memovox/loom/store.py:9`), and every model/storage slot must have a deterministic fallback behind a common interface (spec ┬з7). Today the **model** slots have that registry (`src/memovox/backends/__init__.py`) but the **storage** slots do not тАФ `LoomStore` is a single concrete SQLite class with no interface seam, so an ANN/K├╣zu/Tantivy upgrade can't be dropped in without rewriting callers. This track creates that seam. It also removes the two things that make the free path fall over at scale: `vector_search` recomputes `norm()` on every stored vector for every query (O(N┬╖dim) with two square roots per row), and consolidation re-scans and re-NLIs the *entire* committed-claim set on every pass and silently drops everything past claim #600. Subscriptions (M3.2) and Serving (M3.3) both need an incremental consolidate that only does work proportional to *new* claims and reports what it skipped тАФ this track is the single owner of that capability so they don't each reinvent (and subtly disagree on) the watermark logic.

## Free-path guarantee
- **The four-index store stays SQLite-only and dependency-free.** The ABCs are seams, not new dependencies; the default registry returns the SQLite-backed implementations exactly as today. No numpy: scoring stays in `vectormath.py` (`struct` + Python floats).
- **No-op refactor is byte-identical.** The first commit changes *structure* only тАФ `LoomStore` delegates to `SqliteVectorIndex`/`SqliteLexicalIndex`/`SqliteGraphStore`, but the SQL, ordering, and return tuples are unchanged. The parity gate asserts top-k results (ids **and** scores) are byte-identical before/after.
- **The cosineтЖТdot optimization must not move the byte output.** Unit-normalizing then taking the dot product is mathematically identical to cosine for non-zero vectors, but float reassociation can perturb the last ULP. The parity gate runs the **new** path against a **recorded golden top-k from the old path** and asserts identical ordering; scores are compared with the same rounding the consumer uses (RRF only uses *rank*, not score magnitude тАФ see `rrf_fuse` at `src/memovox/augur/retrieve.py:18`, which keys off `rank`, so rank-stability is the real invariant). The zero-vector guard (`norm(query_vec) == 0.0` at `store.py:557`) and the dim-mismatch skip (`:571`) are preserved verbatim.
- **Everything new defaults OFF.** FTS5 prefilter: default OFF. ANN/K├╣zu/Tantivy selectors: default `"sqlite"` (free). Each new default-OFF flag is added to the **frozen eval-settings snapshot** (W6) so a future default flip can't silently move a gate number.
- **Incremental consolidation is exactly equivalent to a full pass on the free corpus.** `consolidate()` with a cold (unset) watermark must produce the identical graph/report as today's full scan. The incremental path is an optimization that *adds* a watermark cursor; with no watermark it degrades to today's behavior. The INCREMENTAL==FULL gate proves this.
- **Paging replaces silent truncation but does not change committed answers.** On the golden corpus (well under any cap) `claims_skipped==0` and `capped==False`, so behavior is unchanged; the cap only ever engages at synthetic scale, where it is *reported* rather than silent.

## Risks & mitigations
- **Incremental-equivalence is genuinely hard (review-flagged).** Two specifics: (a) the **hub-token quadratic** тАФ a stopword-surviving common token makes the inverted-index candidate set in `find_contradictions` (`:93`тАУ`:108`) blow up, so "new-vs-all" can still be expensive; mitigate by keeping the candidate set drawn from all-committed but the *outer* loop restricted to new claims, and validate cost in `eval/scale.py`, not on the golden corpus. (b) the **`dedup_claims` vs `find_contradictions` cap disagreement** тАФ `dedup_claims` (`:144`) has no cap while the other two truncate at 600 with potentially different orderings; mitigate by unifying both caps onto a single deterministic ordering (W5) and asserting INCREMENTAL==FULL. **The 2тАУ3 video golden corpus will not exercise this** тАФ validate at synthetic scale via `eval/scale.py` (W7).
- **CosineтЖТdot float drift.** Reassociating the sum or normalizing can perturb the last ULP and flip a tie in ranking. Mitigate: the W2 parity gate compares **rank order** (what RRF actually uses тАФ `rrf_fuse` keys off `rank`, `retrieve.py:18`), and ties are broken deterministically by `moment_id`; record the golden top-k from the *pre-optimization* code so the gate is a true tripwire.
- **Migration of existing stores.** Backfilling normalization must be idempotent and one-time (the `vectors_normalized` meta flag), must not touch `visual_vectors` (different space тАФ guarded by the M1.1 space tag), and must survive an interrupted run (re-running re-normalizes only if the flag is unset тАФ but re-normalizing an already-unit vector is a no-op, so it is safe either way).
- **No-op refactor scope creep.** The temptation is to "improve while refactoring." Mitigate: W1 is *purely* structural and parity-locked; the optimization is W3, after the parity gate exists.
- **Single-owner boundary leakage.** M3.2/M3.3 must call the incremental-consolidate primitive, not copy the watermark logic. Mitigate: expose it as one public function with a documented `since_watermark`/scoping parameter and note in the M3.2/M3.3 docs that reimplementation is forbidden.
- **MCP stdout discipline.** The cap event and any scale-harness output must log to **stderr** (MCP speaks JSON-RPC on stdout). Mitigate: route all reporting through the M0.1 stderr logging hook; no stray `print` to stdout in `consolidate.py`/`eval/scale.py`.
- **Frozen-snapshot omission.** A new default-OFF flag not added to the snapshot lets a future default flip silently move a gate. Mitigate: W6 test asserts every new flag is pinned.
