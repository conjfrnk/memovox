# M-X — Cross-cutting hardening & global disciplines

> **Wave:** 0 · **Effort:** M · **Status:** not started
> **Depends on:** folds into the tracks named per item (M1.1, M0.3, M3.2, M1.2, M3.3) · **Owns (single-owner concerns):** the two global eval disciplines — (a) thin-fixture gating, (b) the frozen eval-settings snapshot of default-OFF flags · **Blocks:** none
> **Spec:** §5 (extraction / hybrid retrieval), §9 (throughput, graceful degradation, cost levers), §12 (risks: claim granularity, ColPali cost, ToS / private-by-default)

## Goal
This is a **tracking + discipline** doc, not a single build. The completeness review surfaced a tail of cross-cutting items that have no natural home in one feature track and would otherwise be dropped. Each is given a precise scope and the eval/guard it needs, plus the *exact* track it folds into so it is built once and not duplicated. M-X additionally **owns** the two program-wide eval disciplines every other track must honor: (a) every new metric lands **ungated** and is gated only once ≥3 stable golden items exist, and (b) a **frozen eval-settings snapshot** pins the growing surface of default-OFF flags so a future default flip cannot silently move a gate number. The only first-class standalone build here is **X.4** (structured-JSON extraction output mode); everything else is a folded sub-task tracked from this doc.

## Why it matters
The spec's §9 names the visual track as *the* throughput bottleneck and lists graceful-degradation + cost levers; §12 names claim-granularity tuning, ColPali storage cost, and a ToS / private-by-default posture as standing risks; §5 calls for schema-targeted extraction. None of these is a marquee Phase-4 bullet, so without a single tracking doc they fall through the cracks between feature tracks. The two global disciplines are the load-bearing insurance for the whole roadmap: the dominant program risk (review risk 1) is **eval thinness / gate flakiness** — a hard gate on a 2–3 item fixture is noise, not a signal — and the second risk (review risk 2) is **determinism erosion** as the default-OFF flag surface grows. M-X makes both disciplines explicit, owned, and enforced by a test, so every other track inherits them instead of re-litigating them.

## Scope (reconciled)

**Owned standalone build:**
- **X.4 — Structured-JSON extraction OUTPUT MODE (§5).** A schema-targeted entity/claim extraction *output* — given a Moment (or a video), emit a validated JSON document conforming to a declared entity/claim schema. This is **distinct** from the existing `Answer.to_dict()` / `Synthesis.to_dict()` JSON dump (`augur/types.py:34-40`, surfaced by `cli.py:50`): that dumps a *generated answer*; X.4 dumps the *extracted knowledge structure* (typed claims with subject/predicate/object/type + resolved entities), schema-validated, deterministic on the free path. Small standalone.

**Tracking entries (each folds into a named track; documented here so it is not dropped):**
- **X.1 — Visual-track parallelism (§9 bottleneck) → folds into M1.1** (and consumed by M3.3 serving). The per-keyframe loop in `tessera.run` is serial; parallelize the per-frame work (frame extract → OCR → caption). The only real throughput lever. Measured by M0.1's tracer. *This doc does not re-specify it — M1.1 owns the implementation; M-X tracks it and pins its determinism guard.*
- **X.2 — Reliability bundle → folds into M0.3 / M3.2.** Three §9/§4.1 items: (i) a **persisted visual-backfill flag** on Video/Moment recording "visual layer missing", plus a **targeted re-process** path that re-runs only the visual stage for the flagged video — *not* a whole-video re-ingest; (ii) the **§4.1 ffprobe ASR-readiness gate** — refuse / warn-loud before ASR when `probe()` reports no audio stream; (iii) the **§9 captions-as-fused-prior** cost lever — when source captions exist, use them as a prior to reduce ASR cost rather than re-transcribing blindly.
- **X.3 — Claim-granularity tuning curve + extraction-prompt knob (§12) → folds into M1.2 eval.** A claims-per-moment vs groundedness/salience curve (the diagnostic), and a `min_words` / granularity knob on the extractor so the curve is actionable. *M1.2 owns the eval block; M-X tracks the knob surface.*
- **X.5 — ToS / private-by-default posture (§12/§9) → folds into M3.3.** Beyond M3.3's `local_only` toggle: respect source terms-of-service, a **private-by-default** store posture, and a retention/redaction surface. Policy + small code; M3.3 owns the `local_only`/serving code, M-X tracks the policy and the redaction seam.

**Owned global disciplines (enforced here, inherited by every track):**
- **Discipline (a) — thin-fixture gating.** A new metric lands in the eval report **ungated** (computed, printed, NOT in `_check_thresholds`). It is promoted to a gate **only** once ≥3 stable golden items exercise it. This is the `topic_f1`/`entity_f1`/`der` lesson already encoded in `eval/harness.py:638-649`. M-X documents the rule and adds the *meta-test* that catches an accidental premature gate.
- **Discipline (b) — frozen eval-settings snapshot.** A single pinned snapshot of every default-OFF / soft-default flag (`decay_enabled`, `visual_retrieval`, `colpali_enabled`, `planner_agentic`, `budget_mode`, `otel_enabled`, …) — not just `_FREE_BACKENDS`. A test asserts the live `Settings` defaults equal the snapshot, so flipping a default fails loudly instead of silently moving a gate number. M0.1 establishes the snapshot *mechanism* (`_DEFAULT_OFF_FLAGS` in `eval/harness.py`); **M-X owns the snapshot's completeness** — every track that adds a flag must add it here, and the meta-test proves no flag is unpinned.

**NON-GOALS / deferrals:**
- M-X does **not** implement X.1/X.2/X.3/X.5 — those land inside their host tracks; this doc only specifies their mini-scope + guard and pins their determinism/free-path constraints. Only **X.4** and the **two disciplines** are built under the M-X commits below.
- ANN/Kùzu/Tantivy opt-in backends are **out of scope** and out of any gate (they belong to M0.2's opt-in column).
- A real legal ToS engine is out of scope; X.5 is a posture (default-private store + a redaction seam + a documented policy), not a compliance product.
- Generative-LLM structured extraction is opt-in for X.4; the free path is the deterministic rule-based extractor already present.

## Free-path guarantee
- **No new gate, no default flip.** The two disciplines are *guards*, not features: the frozen-snapshot test pins the **current** default values, so by construction the free path is unchanged — the test fails only if someone *changes* a default. The thin-fixture meta-test only inspects which keys `_check_thresholds` reads; it adds no gate.
- **X.4 free path is the deterministic rule-based extractor.** Structured extraction reuses `_extract_rule_based` (no LLM) by default; the optional generative path is `is_generative`-gated exactly as `extract_claims` already is (`claims.py:165-171`). Output is deterministic JSON (sorted keys, stable claim ids via `make_claim_id`). The new command is **additive** — it touches no existing output: `Answer.to_dict()`, retrieval order, digests, and the four gates stay byte-identical.
- **All folded items keep their host track's free-path contract.** X.1 parallelism defaults to **worker=1 / serial** so `tessera.run` output (the `kept`-ordered events) is byte-identical regardless of pool (M1.1's guarantee; M-X pins it). X.2's ffprobe gate only *adds* a fail-loud path for genuinely-broken input (a no-audio media file) with an `--allow-no-audio` escape, mirroring M0.3's `--allow-cpu`; it never changes a healthy ingest. X.2's backfill flag defaults to a value that makes a healthy video indistinguishable from today. X.3's `min_words` keeps its current default `4`. X.5's private-by-default posture is a *new store's* default; existing stores and the golden corpus are unaffected, and the redaction seam is a no-op unless invoked.
- **Snapshot pins must stay byte-identical:** the snapshot records `visual_enabled=True`, `salience_floor=0.0`, and (as they land) `decay_enabled=False`, `visual_retrieval=False`, `colpali_enabled=False`, `planner_agentic=False`, `budget_mode="soft"`, `otel_enabled=False`, `local_only=False` — the exact values the free path runs with.

## Risks & mitigations
- **Snapshot drift / incompleteness (review risk 2).** A track adds a default-OFF flag and forgets to pin it → a future flip silently moves a gate. *Mitigation:* W1's **reflection-based completeness** test (iterate `dataclasses.fields(Settings)`) forces every new flag to be either pinned or explicitly allow-listed; CI catches an unpinned flag immediately.
- **Premature gating on a thin corpus (review risk 1, the dominant risk).** A future session hard-gates `entity_f1`/`der`/a new metric on 2–3 items → CI flakes. *Mitigation:* W2 encodes `_MIN_FIXTURES_TO_GATE=3` and a meta-test that trips when a gated metric lacks fixtures; the rule auto-enforces once talk_c (M1.2) reaches 3.
- **X.4 confused with the Answer dump.** A reviewer or future caller might re-route X.4 through `Answer.to_dict()`. *Mitigation:* the W3 test explicitly asserts the output is **not** answer-shaped (no `text`/`citations`) — it is the extracted knowledge structure.
- **X.2 ffprobe gate false-positives** on exotic containers where `probe()` mis-reports `has_audio`. *Mitigation:* fail-loud has an `--allow-no-audio` escape (mirrors `--allow-cpu`); transcript-only and healthy paths are untouched; the gate only fires for media sources `probe()` calls audio-less.
- **X.1 parallelism eroding determinism** (review risk on the visual track). *Mitigation:* tracked as default `visual_workers=1` + ordered reassembly; M1.1 owns the byte-identity assertion; concurrency >1 is opt-in and ungated.
- **MCP stdout discipline** (review risk 5). X.4 prints JSON to stdout (a CLI command, correct), but any diagnostic must go to **stderr** — the W3 verify step checks no stray logging pollutes stdout.
- **Tracking-doc rot.** W4 items are specified here but committed elsewhere → they could silently stop matching the host track. *Mitigation:* each W4 item is a Definition-of-Done checkbox in its host track's doc; this doc cites the host track id so the two stay cross-referenced.
